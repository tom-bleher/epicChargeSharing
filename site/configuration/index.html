    Configuration Guide¶

    This document provides a complete reference for all configuration parameters in epicChargeSharing.

    Configuration System¶

    Configuration is managed through include/Config.hh using a hierarchical struct-based system:

    namespace ECS::Config {
    struct DetectorGeometry { ... };
    struct PhysicsParameters { ... };
    struct NoiseModel { ... };
    struct ReconstructionConfig { ... };
    struct GlobalConfig { ... };  // Combined config
}

    Backward Compatibility: The Constants namespace wraps ECS::Config for legacy code:

    namespace Constants {
    inline const G4double DETECTOR_SIZE = 30.0 * mm;
    // etc.
}

    Detector Geometry¶

    DetectorGeometry Struct¶

    Parameter Type Default Unit Description
    detectorSize G4double 30.0 mm Square detector side length
    detectorThickness G4double 0.05 mm Silicon substrate thickness
    worldSize G4double 5.0 cm World volume size
    detectorZPosition G4double -1.0 cm Z position of detector
    pixelSize G4double 0.1 mm Pixel pad side length
    pixelThickness G4double 0.001 mm Pixel pad thickness
    pitch G4double 0.5 mm Pixel center-to-center spacing
    pixelCornerOffset G4double 0.1 mm Distance from detector edge to first pixel
    neighborhoodRadius G4int 2 - Radius for charge sharing grid
    storeFullGrid G4bool true - Store full detector grid data
    geometryTolerance G4double 1.0 µm Geometry overlap tolerance
    precisionTolerance G4double 1.0 nm Numerical precision tolerance

    Geometry Calculations¶

    Pixels per side: 

    pixelsPerSide = (detectorSize - 2 * pixelCornerOffset) / pitch + 1
// Default: (30 - 0.2) / 0.5 + 1 = 60 pixels per side

    Total pixels: 

    totalPixels = pixelsPerSide * pixelsPerSide
// Default: 60 × 60 = 3,600 pixels

    Neighborhood size: 

    neighborhoodSide = 2 * neighborhoodRadius + 1
// Default: 2 × 2 + 1 = 5 (5×5 = 25 pixels)

    Constants Namespace Equivalents¶

    namespace Constants {
    inline const G4double DETECTOR_SIZE = 30.0 * mm;
    inline const G4double DETECTOR_WIDTH = 0.05 * mm;
    inline const G4double PIXEL_SIZE = 0.1 * mm;
    inline const G4double PIXEL_PITCH = 0.5 * mm;
    inline const G4double PIXEL_CORNER_OFFSET = 0.1 * mm;
    inline constexpr G4int NEIGHBORHOOD_RADIUS = 2;
}

    Physics Parameters¶

    PhysicsParameters Struct¶

    Parameter Type Default Unit Description
    ionizationEnergy G4double 3.6 eV Energy per electron-hole pair in silicon
    gain G4double 20.0 - AC-LGAD amplification factor (typical: 8-25)
    d0 G4double 1.0 µm Transverse hit size (Tornago Eq. 4)
    elementaryCharge G4double 1.602×10⁻¹⁹ C Electron charge
    maxStepSize G4double 20.0 µm Maximum step length in silicon
    primaryParticleZ G4double 0.0 cm Z position of particle gun

    Charge Generation¶

    Electron-hole pairs: 

    pairs = (energyDeposit [MeV] × 10⁶) / ionizationEnergy [eV]
// For 10 GeV electron: ~2.8×10⁶ pairs

    Amplified charge: 

    amplifiedPairs = pairs × gain
// With gain 20: ~5.6×10⁷ effective electrons

    Constants Namespace Equivalents¶

    namespace Constants {
    inline constexpr G4double IONIZATION_ENERGY = 3.6;  // eV
    inline constexpr G4double GAIN = 20.0;
    inline constexpr G4double D0 = 1.0;  // µm
    inline constexpr G4double ELEMENTARY_CHARGE = 1.602176634e-19;
    inline const G4double MAX_STEP_SIZE = 20.0 * micrometer;
}

    Noise Model¶

    NoiseModel Struct¶

    Parameter Type Default Description
    gainSigmaMin G4double 0.010 Minimum gain noise σ (1%)
    gainSigmaMax G4double 0.050 Maximum gain noise σ (5%)
    noiseElectronCount G4double 500.0 Electronic noise in electrons
    enablePositionDependence G4bool false Position-dependent noise
    edgeEffectWidth G4double 10.0 Edge effect width (if enabled)
    edgeNoiseMultiplier G4double 1.5 Noise multiplier at edges
    enableCrosstalk G4bool false Enable pixel crosstalk
    crosstalkFraction G4double 0.02 Crosstalk fraction (2%)

    Noise Application¶

    Gain noise (multiplicative, per-pixel): 

    // Each pixel has a fixed σ_gain randomly assigned between min and max
charge_with_noise = charge × (1 + G4RandGauss(0, σ_gain))

    Electronic noise (additive): 

    σ_electronic = noiseElectronCount × elementaryCharge
charge_final = max(0, charge_with_noise + G4RandGauss(0, σ_electronic))

    Constants Namespace Equivalents¶

    namespace Constants {
    inline constexpr G4double PIXEL_GAIN_SIGMA_MIN = 0.010;
    inline constexpr G4double PIXEL_GAIN_SIGMA_MAX = 0.050;
    inline constexpr G4double NOISE_ELECTRON_COUNT = 500.0;
}

    Reconstruction Settings¶

    ReconstructionConfig Struct¶

    Parameter Type Default Description
    signalModel SignalModel LogA Signal sharing model
    reconMethod ReconMethod DPC Position reconstruction method
    denominatorMode DenominatorMode Neighborhood Fraction normalization mode
    dpcKx G4double 0.478 mm DPC k coefficient for x
    dpcKy G4double 0.471 mm DPC k coefficient for y
    dpcTopNPixels G4int 4 Number of pixels for DPC (always 4)
    linearBeta G4double 0.003 LinA attenuation coefficient
    emitDistanceAlpha G4bool false Store d_i and α_i in output
    runFitGaus1D G4bool true Run 1D Gaussian fitting
    runFitGaus2D G4bool false Run 2D Gaussian fitting

    Signal Models¶

    Model Enum Value Formula
    LogA SignalModel::LogA w_i = α_i / ln(d_i/d₀)
    LinA SignalModel::LinA w_i = α_i × exp(-β × d_i)

    Reconstruction Methods¶

    Method Enum Value Description
    LogA ReconMethod::LogA Chi-square fit using LogA model
    LinA ReconMethod::LinA Chi-square fit using LinA model
    DPC ReconMethod::DPC Direct position from 4-pixel charge ratios

    Denominator Modes¶

    Mode Enum Value Normalization
    Neighborhood DenominatorMode::Neighborhood F_i = w_i / Σ_all(w_n)
    RowCol DenominatorMode::RowCol Row: Σ_row(w_n), Col: Σ_col(w_n)
    ChargeBlock DenominatorMode::ChargeBlock F_i = w_i / Σ_4closest(w_n)

    DPC Mode Behavior¶

    When reconMethod = ReconMethod::DPC:

    Feature Behavior
    Full grid storage Disabled (only 4 pixels needed)
    Gaussian fitting Disabled (DPC provides direct position)
    Signal model Uses DPC_SIGNAL_MODEL setting

    Constants Namespace Equivalents¶

    namespace Constants {
    inline constexpr ReconMethod RECON_METHOD = ReconMethod::DPC;
    inline constexpr SignalModel DPC_SIGNAL_MODEL = SignalModel::LogA;
    inline constexpr DenominatorMode DENOMINATOR_MODE = DenominatorMode::Neighborhood;
    inline constexpr G4bool IS_DPC_MODE = (RECON_METHOD == ReconMethod::DPC);
    inline const G4double DPC_KX = 0.478 * mm;
    inline const G4double DPC_KY = 0.471 * mm;
    inline constexpr G4int DPC_TOP_N_PIXELS = 4;
    inline constexpr G4double LINEAR_CHARGE_MODEL_BETA = 0.001;
}

    Output Control¶

    Output Branches by Mode¶

    DENOMINATOR_MODE controls which branches are saved:

    Mode Branches
    Neighborhood Fi, Qi, Qn, Qf
    RowCol FiRow, FiCol, QiRow, QiCol, QnRow, QnCol, QfRow, QfCol
    ChargeBlock FiBlock, QiBlock, QnBlock, QfBlock

    Full Grid Storage¶

    STORE_FULL_GRID (automatically disabled in DPC mode):

    Value Effect
    true Store complete detector grid arrays
    false Store only neighborhood data

    Post-Processing Control¶

    Setting Default Description
    FIT_GAUS_1D !IS_DPC_MODE Run 1D Gaussian fitting
    FIT_GAUS_2D false Run 2D Gaussian fitting

    Runtime Configuration¶

    G4Messenger Commands¶

    Detector Configuration: 

    /ecs/detector/pixelSize <value> <unit>
/ecs/detector/pixelPitch <value> <unit>
/ecs/detector/pixelCornerOffset <value> <unit>

    Particle Gun Configuration: 

    /ecs/gun/fixedPosition <true|false>
/ecs/gun/fixedX <value>
/ecs/gun/fixedY <value>

    Examples¶

    # Set pixel pitch to 400 µm
/ecs/detector/pixelPitch 0.4 mm

# Use fixed position mode
/ecs/gun/fixedPosition true
/ecs/gun/fixedX 0.125 mm
/ecs/gun/fixedY 0.125 mm

    Example Macro Files¶

    Batch Run (macros/run.mac): 

    /control/verbose 0
/run/verbose 0
/event/verbose 0
/tracking/verbose 0

/run/initialize

/gun/particle e-
/gun/energy 10 GeV

/run/beamOn 10000

    Position Sweep: 

    /control/verbose 0
/run/verbose 0
/run/initialize

/gun/particle e-
/gun/energy 10 GeV

# Enable fixed position mode
/ecs/gun/fixedPosition true

# Sweep across positions
/ecs/gun/fixedX 0.0 mm
/ecs/gun/fixedY 0.0 mm
/run/beamOn 1000

/ecs/gun/fixedX 0.125 mm
/ecs/gun/fixedY 0.0 mm
/run/beamOn 1000

# Continue sweep...

    Visualization (macros/vis.mac): 

    /vis/open OGL 800x600-0+0
/vis/drawVolume
/vis/viewer/set/viewpointVector 1 1 1
/vis/viewer/set/autoRefresh true
/vis/scene/add/trajectories smooth
/vis/scene/add/hits
/vis/scene/endOfEventAction accumulate

    High-Resolution Study¶

    // In Config.hh - modify defaults
struct DetectorGeometry {
    G4double pixelSize = 0.05 * mm;       // Smaller pixels
    G4double pitch = 0.25 * mm;           // Tighter pitch
    G4int neighborhoodRadius = 3;         // Larger neighborhood
    G4bool storeFullGrid = true;
};

struct ReconstructionConfig {
    ReconMethod reconMethod = ReconMethod::LogA;  // Chi-square fit
    G4bool runFitGaus1D = true;
    G4bool runFitGaus2D = true;
};

    Fast DPC Analysis¶

    // In Config.hh - DPC mode (default)
struct ReconstructionConfig {
    SignalModel signalModel = SignalModel::LogA;
    ReconMethod reconMethod = ReconMethod::DPC;
    // Full grid and fitters automatically disabled
};

    Changing Configuration¶

    Compile-Time Changes¶

    1. Edit include/Config.hh
    2. Modify struct defaults or Constants namespace values
    3. Rebuild: 
      cd build
make -j$(nproc)

    Runtime Changes (Limited)¶

    Some parameters can be changed via G4Messenger commands before /run/initialize:

    ./epicChargeSharing
/ecs/detector/pixelPitch 0.4 mm
/run/initialize
/run/beamOn 1000

    Note: Most physics parameters require recompilation.

    Quick Reference¶

    Reconstruction Method Selection¶

    Goal Setting
    Fast position reconstruction RECON_METHOD = DPC
    Best resolution (slow) RECON_METHOD = LogA
    Alternative attenuation RECON_METHOD = LinA

    Output Size Control¶

    Goal Settings
    Minimal output STORE_FULL_GRID = false, DENOMINATOR_MODE = Neighborhood
    Full analysis STORE_FULL_GRID = true, emitDistanceAlpha = true
    Row/Col fitting DENOMINATOR_MODE = RowCol

    Noise Level Control¶

    Goal Settings
    Ideal (no noise) gainSigmaMin = gainSigmaMax = 0, noiseElectronCount = 0
    Realistic Use defaults
    High noise study Increase gainSigmaMax, noiseElectronCount
